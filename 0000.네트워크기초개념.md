네트워크 기초 강의의 핵심 개념을 체계적으로 정리해 보겠습니다. IT 전문가로서 **네트워크의 기본 구성 요소와 동작 원리**를 깊이 있게 이해하는 것이 중요합니다.

-----

##  1. 네트워크의 기본 개념

### 1️⃣ 네트워크란 무엇인가?

  * **정의:** 두 대 이상의 컴퓨터나 장치(스위치, 라우터 등)가 **데이터를 주고받을 수 있도록 연결된 시스템**을 의미합니다.
  * **목표:** **자원 공유** (프린터, 파일, 서버 등)와 **정보 교환** (통신)을 가능하게 하는 것입니다.

### 2️⃣ 네트워크 모델: OSI 7 계층과 TCP/IP 4계층

대부분의 네트워크 통신은 계층화된 모델을 따르며, 이는 복잡한 통신 과정을 표준화하고 문제 해결을 용이하게 합니다.

| 계층 번호 (OSI) | OSI 7 계층 | TCP/IP 4계층 | 주요 역할 및 PDU (Protocol Data Unit) |
| :---: | :--- | :--- | :--- |
| 7 | **응용 계층 (Application)** | **응용 계층** | 사용자와 직접 상호작용하며 서비스 제공 (HTTP, FTP, SMTP) - **데이터** |
| 6 | **표현 계층 (Presentation)** | - | 데이터 형식 지정 및 변환, 암호화/복호화 |
| 5 | **세션 계층 (Session)** | - | 통신 세션 수립, 관리, 종료 |
| 4 | **전송 계층 (Transport)** | **전송 계층** | 데이터 전송의 **신뢰성** (TCP) 또는 **효율성** (UDP) 보장 - **세그먼트(Segment)** |
| 3 | **네트워크 계층 (Network)** | **인터넷 계층** | **경로 설정(라우팅)** 및 논리적 주소(IP 주소) 지정 - **패킷(Packet)** |
| 2 | **데이터 링크 계층 (Data Link)** | **네트워크 액세스 계층** | 물리적 주소(MAC 주소) 지정, 오류 감지, 프레임 구성 - **프레임(Frame)** |
| 1 | **물리 계층 (Physical)** | **네트워크 액세스 계층** | 전기적/물리적 신호로 데이터 전송 (케이블, 무선) - **비트(Bit)** |

## 📢 네트워크 통신 방식의 다양한 종류 

네트워크 통신은 데이터를 주고받는 방식과 대상의 수에 따라 여러 가지 방식으로 나뉩니다. 유니캐스트(Unicast) 외에 주요 통신 방식인 **브로드캐스트**, **멀티캐스트**, 그리고 **애니캐스트**를 설명합니다.

---

### 1. 👥 유니캐스트 (Unicast)

가장 흔한 통신 방식으로, **1:1 통신**을 의미합니다.

* **설명**: 출발지에서 지정된 **하나의 목적지**로만 데이터를 전송합니다.
* **특징**: 대부분의 인터넷 트래픽(웹 서핑, 이메일, 파일 전송)이 유니캐스트 방식입니다. 수신을 원하지 않는 장치는 데이터를 처리하지 않습니다.
* **주소**: 출발지 및 목적지 **MAC 주소**와 **IP 주소**가 모두 특정 장치(Host)의 주소로 지정됩니다.

---

### 2. 🌍 브로드캐스트 (Broadcast)

네트워크에 연결된 **모든 장치**에게 데이터를 한 번에 전송하는 방식입니다.

* **설명**: 출발지에서 동일 네트워크(서브넷) 내의 **모든 호스트**에게 데이터를 전송합니다.
* **특징**: 수신을 원하든 원하지 않든, 네트워크상의 모든 장치가 해당 데이터를 수신하고 처리해야 합니다.
    * 라우터에 의해 **브로드캐스트 도메인**이 분리됩니다. 즉, 브로드캐스트 트래픽은 라우터를 넘어가지 못합니다.
* **주소**: 목적지 MAC 주소는 `FF:FF:FF:FF:FF:FF` (All F's)로, 목적지 IP 주소는 **브로드캐스트 주소**로 지정됩니다.
* **사용 예**: **ARP(Address Resolution Protocol)** 요청, DHCP(IP 주소 할당) 요청 등이 대표적입니다.

---

### 3. 📡 멀티캐스트 (Multicast)

특정 그룹에 속한 **다수의 장치**에게만 데이터를 효율적으로 전송하는 방식입니다. **1:N 통신**을 지원합니다.

* **설명**: 출발지에서 데이터를 전송하면, 해당 데이터를 수신하겠다고 미리 등록한 (**그룹에 가입한**) 장치들에게만 전송됩니다.
* **특징**: 브로드캐스트처럼 불필요한 장치에게 데이터를 보내지 않아 효율적이며, 유니캐스트처럼 수신자 수만큼 복사본을 만들 필요가 없어 대역폭을 절약합니다.
* **주소**: **멀티캐스트 IP 주소**(`224.0.0.0` ~ `239.255.255.255` 범위)와 이에 매핑되는 **멀티캐스트 MAC 주소**를 사용합니다.
* **사용 예**: IPTV (실시간 스트리밍), 화상 회의, 라우팅 프로토콜(OSPF, EIGRP)의 정보 교환.

---

### 4. 🔀 애니캐스트 (Anycast)

동일한 IP 주소를 가진 **가장 가까운(가장 빠른) 하나의 장치**로만 데이터를 전송하는 방식입니다.

* **설명**: 출발지에서 데이터를 전송하면, 동일한 IP 주소를 공유하는 여러 서버 중 **네트워크적으로 가장 가까운** 서버에게만 데이터가 전달됩니다. 이는 **1:1-of-N 통신**으로 간주됩니다.
* **특징**: 라우팅 프로토콜이 가장 효율적인 경로를 결정하여 트래픽을 분산시키고 부하를 줄입니다.
* **사용 예**: **DNS 루트 서버**나 **CDN(Content Delivery Network)** 서버와 같이 전 세계에 분산 배치된 서비스에 주로 사용됩니다. 사용자가 어느 위치에 있든 가장 가까운 서버로부터 빠르게 응답을 받을 수 있게 합니다.

---

### 📚 통신 방식별 비교 요약

| 통신 방식 | 전송 대상 | 목적지 주소 | 효율성 | 주요 사용처 |
| :---: | :---: | :---: | :---: | :--- |
| **유니캐스트** | 특정 **1개**의 장치 | 개별 MAC/IP 주소 | 높음 | 웹, 이메일, 파일 전송 |
| **브로드캐스트** | **동일 네트워크 내 모든** 장치 | 브로드캐스트 MAC/IP 주소 | 낮음 (네트워크 부하 유발) | ARP 요청, DHCP 요청 |
| **멀티캐스트** | **그룹에 가입된 다수**의 장치 | 멀티캐스트 MAC/IP 주소 | 매우 높음 | IPTV, 화상 회의, 라우팅 정보 교환 |
| **애니캐스트** | **가장 가까운 1개**의 장치 (N개 중) | 동일 IP 주소 | 매우 높음 | DNS 루트 서버, CDN |

이러한 통신 방식들은 네트워크 장비와 프로토콜에 의해 구별되어 처리되며, 특히 **멀티캐스트와 애니캐스트는 대규모 분산 환경**에서 서비스 품질과 효율성을 결정하는 중요한 요소입니다.

##  2. 핵심 주소 체계: IP와 MAC

### 1️⃣ IP 주소 (Internet Protocol Address)

  * **역할:** 네트워크 계층에서 사용되는 **논리적 주소**로, 통신하고자 하는 **네트워크 경로를 식별**합니다. 라우터가 라우팅(경로 지정)할 때 사용됩니다.
  * **종류:**
      * **IPv4:** 32비트 주소 체계 ($xxx.xxx.xxx.xxx$ 형식). 현재 주소 고갈 문제로 인해 **NAT** 기술이 필수적으로 사용됩니다.
      * **IPv6:** 128비트 주소 체계로, IPv4의 고갈 문제를 해결하기 위해 도입되었습니다.
  * **공인 IP vs. 사설 IP:**
      * **공인 IP (Public IP):** 전 세계적으로 **유일**하며, 인터넷에서 라우팅 가능한 주소입니다.
      * **사설 IP (Private IP):** 내부 네트워크에서만 사용되며, **RFC 1918**에 정의된 특정 대역($10.x.x.x$, $172.16.x.x$\~$172.31.x.x$, $192.168.x.x$)에 속합니다.

### 2️⃣ MAC 주소 (Media Access Control Address)

  * **역할:** 데이터 링크 계층에서 사용되는 **물리적 주소**로, 네트워크 카드(NIC)에 고정되어 있는 **하드웨어 주소**입니다. 동일 세그먼트 내에서 장치를 식별합니다.
  * **특징:** 48비트(6바이트)로 구성되며, 이더넷 통신 시 최종 목적지를 찾는 데 사용됩니다.

##  3. 주요 네트워크 장비

네트워크 장비는 통신을 연결하고 데이터를 올바른 경로로 전달하는 역할을 합니다.

| 장비 | 계층 | 주요 역할 |
| :--- | :--- | :--- |
| **허브 (Hub)** | 1계층 (물리) | 단순히 신호를 증폭하고 모든 포트로 재전송합니다. **충돌** 발생 가능성이 높습니다. |
| **스위치 (Switch)** | 2계층 (데이터 링크) | MAC 주소를 기반으로 프레임을 목적지 포트로만 전달합니다. **충돌 도메인을 분할**하여 효율성을 높입니다. |
| **라우터 (Router)** | 3계층 (네트워크) | **IP 주소**를 기반으로 **최적의 경로**를 찾아 패킷을 전송합니다. **네트워크를 연결**하고 브로드캐스트 도메인을 분리합니다. |
| **방화벽 (Firewall)** | 3\~7계층 | 미리 정의된 규칙에 따라 네트워크 트래픽을 감시하고 접근을 허용/차단하여 **보안**을 담당합니다. |

##  4. 핵심 프로토콜

### 1️⃣ TCP (Transmission Control Protocol)

  * **특징:** **신뢰성 있는 연결 지향형 프로토콜**입니다. 데이터를 정확하게 순서대로 전달하며, **3-way Handshake**를 통해 연결을 수립하고 **흐름 제어, 오류 제어**를 수행합니다.
  * **사용 예:** 웹 통신(HTTP), 파일 전송(FTP), 이메일(SMTP) 등 데이터 무결성이 중요한 서비스.

### 2️⃣ UDP (User Datagram Protocol)

  * **특징:** **비연결 지향형 프로토콜**입니다. 신뢰성보다는 **속도와 효율성**을 우선하며, 패킷 손실이나 순서 변경을 신경 쓰지 않습니다.
  * **사용 예:** 실시간 스트리밍, 음성 통화(VoIP), DNS 쿼리 등 빠른 전송이 중요한 서비스.

### 3️⃣ 기타 중요 프로토콜

  * **ARP (Address Resolution Protocol):** IP 주소를 통해 MAC 주소를 알아내는 데 사용됩니다.
  * **DNS (Domain Name System):** 사람이 읽을 수 있는 도메인 이름(예: https://www.google.com/search?q=google.com)을 IP 주소로 변환합니다.

##  계층별 역할 심화 설명

### 4계층: 전송 계층 (Transport Layer)

* **L4 스위치/로드 밸런서:** 이 장비는 TCP/UDP 포트 번호($4$계층 정보)를 기반으로 트래픽을 분산하여 서버의 부하를 균등하게 나누는 데 사용됩니다. 이는 **서비스의 효율성**과 **가용성**을 높이는 핵심 장비입니다.
* **TCP (Transmission Control Protocol):** "연결 지향" 방식으로, 데이터 전송 전에 **3-way Handshake**를 통해 연결을 수립하고, 데이터 손실이나 순서 오류 발생 시 재전송을 요청하는 등 **신뢰성**을 보장합니다.
* **UDP (User Datagram Protocol):** "비연결 지향" 방식으로, 신뢰성은 낮지만 오버헤드가 적어 빠릅니다. 스트리밍, 실시간 게임 등 **속도**가 중요한 서비스에 주로 사용됩니다.

### 3계층: 네트워크 계층 (Network Layer)

* **라우터 (Router):** 네트워크의 핵심 장비로, **IP 주소**를 기반으로 수많은 네트워크 중에서 **최적의 경로**를 결정하고 패킷을 전달하는 **라우팅** 기능을 수행합니다.
* **IP (Internet Protocol):** 데이터를 패킷 단위로 묶어 최종 목적지까지 전달하는 **논리적 주소 체계**입니다. 라우터가 라우팅을 수행하는 유일한 근거가 됩니다.
* **ARP (Address Resolution Protocol):** 3계층의 **IP 주소**를 2계층의 **MAC 주소**로 변환하는 데 사용되는 프로토콜입니다.

### 2계층: 데이터 링크 계층 (Data Link Layer)

* **스위치 (Switch):** **MAC 주소** 테이블을 사용하여 들어온 데이터를 해당 목적지가 연결된 포트로만 전달합니다. 허브와 달리 **불필요한 트래픽을 방지**하여 네트워크 효율성을 크게 향상시킵니다.
* **Ethernet (이더넷):** LAN 환경에서 가장 널리 사용되는 표준 프로토콜로, 물리적인 전송 방식과 MAC 주소 지정 방식을 정의합니다.

네트워크를 이해하는 것은 결국 이 7개의 계층에서 데이터가 어떻게 변환되고, 어떤 장비를 통해 전달되며, 어떤 프로토콜이 약속을 지키는지 파악하는 것입니다. 이 계층 구조는 **보안 영역**에서도 방화벽이나 침입 탐지 시스템(IDS)이 어떤 계층의 정보를 분석하는지를 이해하는 데 매우 중요합니다.

---

## 🗄️ 네트워크 장비, NIC 및 케이블 분류표

 네트워크 통신은 데이터를 전송하는 **물리적 매체(케이블)**, 통신 능력을 부여하는 **인터페이스(NIC)**, 그리고 트래픽을 처리하고 경로를 결정하는 **장비(Device)**의 세 가지 핵심 요소로 구성됩니다.

다음은 이 요소들을 OSI 7계층 모델에 따라 분류하고 주요 역할을 요약한 표입니다.

---

### 1. 🔍 네트워크 장비/구성 요소

| 구성 요소 | OSI 계층 | 데이터 단위 | 주요 역할 |
| :---: | :---: | :---: | :--- |
| **케이블 (UTP, 광섬유)** | 1계층 (물리) | 비트 (Bit) | 데이터의 **물리적 전송 경로** 제공 |
| **NIC** (네트워크 인터페이스 카드) | 1계층/2계층 | 비트 / 프레임 | **PC에 통신 능력 부여**, MAC 주소 보유, 신호 변환 |
| **리피터, 허브** | 1계층 (물리) | 비트 (Bit) | 신호 증폭/재생, 단순 연결 |
| **스위치 (Switch)** | 2계층 (데이터 링크) | 프레임 (Frame) | **MAC 주소** 기반 통신, 충돌 도메인 분리 |
| **라우터 (Router)** | 3계층 (네트워크) | 패킷 (Packet) | **IP 주소** 기반 경로 결정, 네트워크 간 연결 |
| **L3 스위치** | 3계층 (네트워크) | 패킷 (Packet) | 라우터 기능 + 스위치 속도, 대규모 내부 라우팅 |
| **방화벽, 게이트웨이** | 3~7계층 | 패킷/데이터 | 보안 정책 적용, 프로토콜 변환 및 접근 제어 |
| **로드 밸런서** | 4~7계층 | 세그먼트/세션 | 트래픽 분산, 서버 가용성 및 성능 유지 |

---

### 2. 📝 구성 요소별 상세 설명

#### 🔸 1계층 요소: 물리적 전송 및 인터페이스

| 구성 요소 | 설명 |
| :--- | :--- |
| **케이블 (UTP/광섬유)** | **데이터가 이동하는 통로**입니다. UTP(Unshielded Twisted Pair)는 구리선을 사용하며 LAN에서 가장 일반적입니다. **광섬유 케이블**은 빛 신호를 사용하여 장거리 및 고속 전송이 필요할 때 사용됩니다. |
| **NIC (Network Interface Card)** | 컴퓨터에 고유한 **MAC 주소**를 부여하고, 컴퓨터의 디지털 데이터를 케이블을 통과할 수 있는 **전기 또는 광 신호**로 변환하는 장치입니다. |
| **허브/리피터** | **리피터**는 약해진 신호를 증폭합니다. **허브**는 연결된 모든 장치에 신호를 브로드캐스팅하는 단순 연결 장치입니다. |

#### 🔹 2계층 장비: 로컬 통신 및 분할

| 장비 | 설명 |
| :--- | :--- |
| **스위치 (Switch)** | **MAC 주소**를 학습하여 데이터를 원하는 목적지 포트로만 전송합니다. 브로드캐스팅을 제한하고 각 포트를 **독립적인 충돌 도메인**으로 만들어 네트워크 성능을 크게 향상시킵니다. |

#### 🔸 3계층 장비: 경로 결정 및 네트워크 간 연결

| 장비 | 설명 |
| :--- | :--- |
| **라우터 (Router)** | **IP 주소**를 기반으로 패킷이 서로 다른 네트워크를 넘어 목적지까지 도달할 수 있는 **최적의 경로**를 계산하고 전달합니다. **브로드캐스트 도메인**을 분리하여 광범위한 통신을 가능하게 합니다. |
| **L3 스위치** | 고속으로 L3 라우팅을 수행하는 장비로, 대규모 네트워크에서 빠른 속도의 내부 라우팅을 위해 사용됩니다. |

#### 🔹 4~7계층 장비: 보안 및 서비스 관리

| 장비 | 설명 |
| :--- | :--- |
| **방화벽 (Firewall)** | 사전에 정의된 규칙(IP, 포트, 프로토콜)에 따라 트래픽을 검사하고 차단하여 내부 네트워크를 보호하는 보안 장비입니다. |
| **로드 밸런서** | 서비스의 부하를 여러 서버에 효율적으로 나누어 주어, 서비스의 중단 없이 안정적인 운영을 가능하게 합니다. |

## 🛡️ 네트워크 장비와 해킹 방어 전략 연관성 분석

네트워크 장비는 단순한 데이터 전송 도구를 넘어, 해커의 공격을 감지하고 방어하는 **보안 경계선** 역할을 수행합니다. 각 계층별 장비가 해커의 공격에 어떻게 대응하는지 설명해 드리겠습니다.

---

### 1. 🥈 2계층 장비: 스위치와 MAC/ARP 공격 방어

스위치(L2)는 내부 네트워크 통신의 근간이며, 주로 **MAC 주소 기반의 공격**에 취약합니다.

* **관련 공격**: **ARP 스푸핑 (ARP Spoofing)**, **MAC 플러딩 (MAC Flooding)**
    * **ARP 스푸핑**: 공격자가 스위치에 잘못된 MAC-IP 매핑 정보를 전송하여 트래픽을 가로채는 중간자(MiTM) 공격입니다.
    * **MAC 플러딩**: 스위치의 MAC 주소 테이블(CAM Table) 용량을 초과하는 위조된 MAC 주소를 대량 전송하여, 스위치가 허브처럼 작동(Fail Open)하게 만들어 모든 트래픽을 엿듣게 하는 공격입니다.
* **보안 전략 (스위치 기능 활용)**:
    * **Port Security**: 특정 포트에 할당될 수 있는 MAC 주소의 개수를 제한하거나, 특정 MAC 주소만 허용하여 MAC 플러딩 및 스푸핑 공격을 방지합니다.
    * **DHCP Snooping / DAI (Dynamic ARP Inspection)**: DHCP 서버의 정당한 IP 할당 정보를 검증하여, ARP 응답의 유효성을 확인하고 ARP 스푸핑을 차단합니다.

---

### 2. 🥉 3계층 장비: 라우터와 네트워크 경계 보호

라우터(L3)는 네트워크 간의 경계에 위치하며, **IP 주소 기반의 경로 조작**이나 **네트워크 자원 고갈 공격**을 방어합니다.

* **관련 공격**: **IP 스푸핑 (IP Spoofing)**, **DoS/DDoS 공격**
    * **IP 스푸핑**: 패킷의 출발지 IP 주소를 위조하여 방화벽 규칙을 우회하거나, DoS 공격의 발신지를 숨기는 데 사용됩니다.
    * **DoS/DDoS**: 대량의 패킷을 전송하여 라우터의 회선 대역폭이나 패킷 처리 능력을 마비시킵니다.
* **보안 전략 (라우터 기능 활용)**:
    * **Access Control List (ACL)**: 출발지 및 목적지 IP 주소, 포트 번호 등을 기반으로 패킷의 통과 여부를 설정하여 불필요한 트래픽이나 위조된 IP를 가진 패킷(**IP 스푸핑**)을 필터링합니다.
    * **Rate Limiting**: 특정 종류의 트래픽이나 특정 출발지로부터 오는 트래픽의 전송 속도를 제한하여 DoS 공격의 영향을 줄입니다.

---

### 3. 🛡️ 4~7계층 장비: 방화벽, IPS/IDS 및 로드 밸런서

이들 장비는 단순한 전송을 넘어 **트래픽의 내용과 세션을 분석**하여 능동적으로 해커의 공격을 방어합니다.

| 장비 | 주요 방어 역할 | 관련 해커 공격 |
| :---: | :--- | :--- |
| **방화벽 (Firewall)** | **접근 제어 및 망 분리**. L3/L4 정보뿐만 아니라 (차세대 방화벽의 경우) L7 정보까지 분석하여 허가되지 않은 프로토콜의 통신을 차단합니다. | 포트 스캐닝, 불필요한 포트 접근, 특정 프로토콜 이용 공격 |
| **IPS/IDS** | **능동적/수동적 침입 탐지 및 차단**. 이미 알려진 공격 패턴이나 비정상적인 트래픽 흐름을 실시간으로 감지하고 차단합니다. | 웜/바이러스 전파, 버퍼 오버플로우, SQL 인젝션, 악성 코드 다운로드 시도 |
| **로드 밸런서** | **서비스 가용성 확보 및 DDoS 방어**. L7 레벨에서 비정상적인 세션 수립 시도를 걸러내고, 트래픽을 분산시켜 특정 서버의 자원 고갈을 막아 서비스가 중단되지 않게 합니다. | L7 DDoS 공격 (HTTP Flood 등), Slowloris 공격 |

* **강조**: 현대의 방어 전략은 단일 장비가 아닌, **라우터(경계 보호) → 방화벽(접근 제어) → IPS(공격 차단) → 스위치(내부 분리)**로 이어지는 **다중 방어 체계(Defense-in-Depth)**를 구축하는 데 중점을 둡니다.
